# PLANS.md

# Java 版设备日均资费计算器开发计划

## 一、总目标

使用 Java 21 + JavaFX + Maven 开发一个 Windows 桌面软件：

```text
设备日均资费计算器 / Daily Cost Calculator
```

软件用于管理个人设备、一次性附属配件、日均摊销、目标换新周期、统计分析、模型图表、批量导入导出和本地 JSON 数据持久化。

最终交付目标：

```text
可以在 IDEA 中运行
可以通过 mvn test 完成测试
可以通过 package-windows.bat 打包为 Windows exe 安装包
安装后可以离线运行
重新打开软件后数据不丢失
```

---

## 二、阶段开发原则

请严格按阶段开发。

```text
运行 mvn test

```

#### 1.3 计算服务

实现：

```text

[x] StatisticsService
[x] TargetPlanService
[x] FormatService 或 FormatUtil
```

核心公式必须包含：

```text
[x] 已使用天数 = 当前日期 - 购买日期 + 1，最小 1
[x] 设备本体日均 = 设备本体价格 / 设备本体已使用天数
[x] 配件日均 = 配件价格 / 配件已使用天数
[x] 设备当前总日均 = 设备本体日均 + 配件合计日均
[x] 设备总投入 = 设备本体价格 + 配件价格总和
[x] 全部累计投入 = 所有设备总投入之和
[x] 合计日均 = 所有设备当前总日均之和
[x] 等效月租 = 合计日均 × 30.4167
[x] 等效年化成本 = 合计日均 × 365
[x] 加权使用天数 = 设备总投入 / 当前总日均
[x] 目标所需总天数 = 设备总投入 / 目标日均
[x] 还需天数 = 目标所需总天数 - 当前加权使用天数
[x] 预计达成日期 = 当前日期 + 还需天数
[x] 30天下降 = 当前日均 - 设备总投入 / (当前加权使用天数 + 30)
[x] 边际下降/天 = 当前日均 - 设备总投入 / (当前加权使用天数 + 1)
[x] 替换指数 = 目标日均 / 当前日均 × 100，并限制到 1-99 或已达成 100
[x] 建议文本生成
```

## 十二、开发注意事项

1. 不要牺牲计算正确性追求界面效果。
2. 所有金额核心计算必须使用 BigDecimal。
3. UI 层只负责展示和交互，不要直接写复杂公式。
4. 计算结果通过 service 层生成 snapshot 后给 UI 使用。
5. 导入导出、文件读取、备份恢复必须处理异常。
6. 任何用户可能看不懂的错误，都要转成清晰中文提示。
7. 不要联网。
8. 不要使用数据库服务器。
9. 不要要求管理员权限。
10. 不要把 AppData 路径写死。
11. 每完成一个阶段必须运行测试。
12. 如果某阶段失败，先修复，不要继续堆后续功能。
